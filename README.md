# 팀아온다 🌿

> **Team + 아니온듯 다녀갑니다** — 흔적 없이 자연을 즐기는 백패킹 크루 홈페이지

모바일 최적화 정적 웹앱. 순수 HTML/CSS/JS (빌드 불필요).

## 기능
- 🎯 **장비 매칭** — 성향·박지·무게·자차·예산 선택 → 나에게 맞는 텐트·박지 추천
- ⛺ **텐트 비교** — 무게·가격·계절, 키워드 필터(가성비/삼계절/사계절/1~2kg/UL)
- 🔥 **할인·블랙프라이데이** 캘린더
- 📍 **박지 가이드** — 섬·오지·캠핑장, 자차 없이 갈 수 있는 곳 표시
- ✅ 패킹 체크리스트

## 실행
```bash
bash run.sh   # → http://localhost:8020
```

## 데이터 수정
장비·박지·할인·후기는 모두 [`data.js`](data.js) 배열만 고치면 됩니다.
텐트는 `TENT_MODELS`(모델라인) + `VERIFIED`(실측 스펙) 로 관리하며, 브랜드·인원 변형은 자동 생성됩니다.

> 텐트 스펙 중 **✓실측(17종)** 은 웹 조사로 확인한 값, **≈추정** 은 대표값에서 인원별 계산한 값(무게 ±10~15%·가격 ±20% 오차 가능)입니다.

## 후기 전체공유 백엔드 켜기 (Supabase, 무료)
기본은 후기가 **작성 기기에만** 저장됩니다. 아래를 설정하면 **크루 전체가 공유**하는 방명록이 됩니다.

1. [supabase.com](https://supabase.com) 가입 → 새 프로젝트 생성(무료).
2. 좌측 **SQL Editor** 에서 아래 실행:
   ```sql
   create table reviews (
     id bigint generated by default as identity primary key,
     created_at timestamptz default now(),
     name text not null,
     spot text, visited text, gear text,
     stars int check (stars between 1 and 5),
     tags text[] default '{}',
     text text not null
   );
   alter table reviews enable row level security;
   create policy "anyone can read"   on reviews for select using (true);
   create policy "anyone can insert" on reviews for insert with check (true);
   ```
3. **Project Settings → API** 에서 `Project URL` 과 `anon public` key 복사.
4. [`data.js`](data.js) 의 `const SUPABASE` 에 붙여넣기:
   ```js
   const SUPABASE = { url: "https://xxxx.supabase.co", key: "eyJhbGci..." };
   ```
5. 커밋·푸시하면 끝. (anon key는 공개돼도 되는 키입니다. RLS로 읽기·쓰기만 허용됨)
